#  SecureSense
> Healthcare IoT Security Intelligence Agent




##  Problem

Healthcare IoT devices are life-critical but constantly targeted by cyberattacks. Traditional security tools lack clinical context — treating a pacemaker (98/100 criticality) the same as a smart thermostat (20/100 criticality). Isolating a device blindly to stop an attack can cause fatal patient hazards.

Critical clinical assets monitored:
- Pacemaker Controller (98/100 criticality, Patient Multiplier: 1.0)
- Ventilator (97/100 criticality, Patient Multiplier: 1.0)
- ICU Monitor (95/100 criticality, Patient Multiplier: 0.95)
- Infusion Pump (90/100 criticality, Patient Multiplier: 0.90)



## Solution

SecureSense is an autonomous Healthcare SOC Assistant built on the **Google Agent Development Kit (ADK) 2.0** that:
**Detects threats** using a trained XGBoost ML model on the `CIC-IoT-2023` dataset.
**Audits compliance** automatically against NIS2 Article 21/23 and GDPR Article 32/33 controls.
**Maps techniques** to the MITRE ATT&CK framework (T1498 DDoS, T1557 MITM, T1190 Injection).
**Scores clinical risk** using a composite formula that assesses patient safety impact.

##  Setup

```bash
# 1. Clone repo
git clone https://github.com/mobarak93/SecureSense.git
cd SecureSense

# 2. Install dependencies & configure key
uv sync
cp .env.example .env  # Add GEMINI_API_KEY

# 3. Add ML model files to models/ folder
# models/xgboost_ciciot23_class_weight.json, cic_preprocessor.pkl, cic_label_encoder.pkl

# 4. Verify agent routing & run test suite
uv run python verify_agents.py
uv run pytest

### Required External Files

The following files are **not included** in this repository due to their size or other restrictions.

Place them inside the `models/` directory before running SecureSense.

|            File                |            Purpose            |
|----------------------------------------------------------------
| `xgboost_ciciot23_class_weight.json` | Trained XGBoost IDS model |
| `cic_preprocessor.pkl` | Feature preprocessing pipeline |
| `cic_label_encoder.pkl` | Attack class label encoder |

> **Note:** These artifacts can be generated by retraining the model on the CIC-IoT-2023 dataset or obtained from the project maintainer.

## Repository Notes

To keep the repository lightweight and secure, the following files and directories are intentionally **excluded from version control** (`.gitignore`).

| File / Directory           |          Reason                                                         
| -------------------------- | ------------------------------------------------------------------- 
| `.env`                     |Contains  sensitive environment variables (e.g.,GEMINI_API_KEY`).                            |
| `.venv/`                   | Local Python virtual environment. Generate locally using `uv sync`.             
| `models/`                  | Stores trained XGBoost model artifacts. Not included due to size.        
| `.pytest_cache/`           | Automatically generated by pytest.                      
| `security.log`             | Runtime security log generated during execution.                   
| `deployment_metadata.json` | Local deployment metadata generated at runtime.                     

### Required Before Running

Create or provide the following before running SecureSense:

* Configure `.env` with your `GEMINI_API_KEY`.
* Place the required trained model artifacts inside the `models/` directory:

  * `xgboost_ciciot23_class_weight.json`
  * `cic_preprocessor.pkl`
  * `cic_label_encoder.pkl`

All other files and directories are generated automatically during installation or runtime.
